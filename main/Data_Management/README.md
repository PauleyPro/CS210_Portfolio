# Data Management

This folder contains the grocery frequency-processing logic and the files that support it.

## Responsibilities

- `GroceryTracker.cpp` reads the grocery purchase list, stores item counts in the tracker map, and provides the processed frequency data to the menu stages.
- `CS210_Project_Three_Input_File.txt` is the input source used to build the frequency data.
- `frequency.dat` is the backup output generated by the program after the input file is processed.

## File Flow

1. `main.cpp` creates a `GroceryTracker` object.
2. The tracker loads data from `Data_Management/CS210_Project_Three_Input_File.txt`.
3. The tracker writes the processed counts to `Data_Management/frequency.dat`.

## Design Notes

Separating the data-processing files from the menu code keeps the program easier to review and supports the assignment requirement that the frequency data be backed up automatically.
